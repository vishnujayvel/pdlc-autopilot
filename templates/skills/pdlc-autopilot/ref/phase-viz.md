# Phase Visualization

Pure markdown/unicode visualization for PDLC progress. No external tools, no Python вҖ” just text the Director renders inline.

---

## Phase Markers

| Symbol | Meaning |
|--------|---------|
| вң… | Done |
| рҹ”ө | Active (currently executing) |
| в¬ң | Upcoming |
| вқҢ | Failed |
| вҸӯпёҸ | Skipped |

## Progress Bars

```text
Full:    [в–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳ] 8/8
Partial: [в–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘] 4/8
Empty:   [в–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘в–‘] 0/8
```

Use в–Ҳ (full) and в–‘ (empty). 16 chars total width. Scale to batch count.

## Rendering Rules

1. **Use raw text** вҖ” no code fences around the box (render it directly in conversation)
2. **Box width adapts** вҖ” fit content, don't pad excessively
3. **Unicode box drawing** вҖ” `в”Ңв”Җв”җв”Ӯв””в”Җв”ҳв”ңв”Ө` for borders (NOT ASCII `+--+`)
4. **Keep compact** вҖ” one pipeline per render, not multiple stacked
5. **Emoji in markers only** вҖ” вң…рҹ”өв¬ңвқҢвҸӯпёҸ for phases, nowhere else in the box
